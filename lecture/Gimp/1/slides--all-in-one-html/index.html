1. Introduction

ContentsUpPrevNext
Do it with the GIMP → Introduction
  • The GIMP - The GNU Image Manipulation Program
  • A sophisticated image processing and composition program, up-to-par with commercial offerings.
  • Available for UNIXes and other platforms.
  • This Lecture aims to give a taste of the GIMP's capabilities.
  • It is not a comprehensive coverage of them.
  • I will also try to explain some of the algorithms that are used behind the scenes.

1.1. History of the GIMP

ContentsUpPrevNext
Do it with the GIMP → Introduction → History of the GIMP

  • A document that details the GIMP's history can be found here:

    http://www.gimp.org/~sjburges/gimp-history.html

  • GIMP was started as a project by two Berkeley students called Spencer Kimball and Peter Mattis. It was first meant as a university project that they received credit for.
  • GIMP used Motif at first but Kimball and Mattis later developed Gtk+, so it can be used as its underlying GUI library instead. Since then, Gtk+ has become the basis for many other projects including GNOME.
  • GIMP 1.0 was released several years after the switch to Gtk+, and GIMP 1.2 (which already used Gtk+ 1.2) was released three years afterwards.
  • Since then, the GIMP has acclaimed many fans and it is considered the most complete and active image manipulation programs for UNIXes to-date.

1.2. Future Directions

ContentsUpPrevNext
Do it with the GIMP → Introduction → Future Directions

  • The development plan for the future was outlined in the following message:

    "RFC: The Future of the GIMP"

  • GIMP 1.4, which will be developed in the 1.3.x branch, will have the GUI separated from the core, will be ported to Gtk+ 2.0 and will have several other relatively minor enhancements.
  • GIMP 2.0 will have CMYK support, 16-bits per color value support, and many other features. It will use the newly developed GEGL library as the basis for its core.

2.1. Colour Modes

ContentsUpPrevNext
Do it with the GIMP → Overview of Features → Colour Modes
  • Gimp supports RGB, Grayscale and Indexed (= palette based) Images
  • It can convert between them, but conversion from RGB to something else is not reversible.

Demo:

  1. Open lena_color.jpg.
  2. Select <Image> -> Image -> Mode -> Grayscale
  3. Select <Image> -> Image -> Mode -> RGB
  4. As can be seen, the image remained in shades of gray. However, now colours can be added.
  • Some effects are not available in all modes
  • It is recommended to work in RGB or in Grayscale and convert it to be a palette-based one only before publishing it. (and keep a copy of the RGB source)

ContentsUpPrevNext

Demo:

  2. Select the "Color Picker" tool from the toolbox.
  3. Click on pixels of the image (or drag while holding the mouse button) and see the colour values that exist there.

  • It will be integrated into the main source tree in GIMP 2.0.

2.3.2. What are RGB and HSV?

ContentsUpPrevNext
Do it with the GIMP → Overview of Features → RGB and HSV modes → What are RGB and HSV?

RGB

  • RGB stands for Red-Green-Blue.
  • They are additive light components from which one can form every visible shade.

HSV

  • H = Hue : A unit that specifies the "colour" of the colour
  • S = Saturation : Specifies how much black, white or gray is mixed with the colour.
  • V = Value : Corresponds to Brightness.

2.3.3. What about CMYK?

ContentsUpPrevNext
Do it with the GIMP → Overview of Features → RGB and HSV modes → What about CMYK?
  • CMYK stands for Cyan, Magenta, Yellow and blacK.
  • It is a subtractive method that is akin to mixing paint colours.
  • CMYK support will be added in GIMP 1.4.x or 2.0.x, but is not available at the moment.

2.7.1. Alpha

ContentsUpPrevNext
Do it with the GIMP → Overview of Features → Layers → Alpha

Every layer contains a special layer called the "alpha layer" that specifies its opacity. This layer is treated as a black and white bitmap that can be manipulated with all the normal tools, and in which black is considered full transparency (zero opacity) and white is full opacity between this layer and the layer below it.

The Resultant pixel of the two layers is calculated using the formula:

Result = (1-alpha) * PixelBelow + alpha * PixelAbove

ContentsUpPrevNext
Do it with the GIMP → Overview of Features → Layers → Alpha → Demo 1

Demo

  2. Duplicate the layer using <Layers> -> Duplicate Layer .
  3. Render a nice gradient on the lower layer.
  4. Generate an alpha on the upper layer by using <Layers> -> "Add Layer Mask..." .
  5. Select the paintbrush tool and paint black strokes on the alpha of the upper layer.
  6. Switch to a grey colour and paint grey strokes.
  7. Finalize the Layer's Mask using <Layers> -> "Apply Layer Mask"

3.1. Brightness and Contrast

ContentsUpPrevNext
Do it with the GIMP → LUT (Look-Up Tables Effects) → Brightness and Contrast

Question

How can we increase/decrease the brightness and contrast of an image?

3.1.1. Answer

ContentsUpPrevNext
Do it with the GIMP → LUT (Look-Up Tables Effects) → Brightness and Contrast → Answer

Increasing or decreasing the brightness involves adding or subtracting a certain amount from the value of each pixel.

The contrast can be increased by plotting an y=mx graph according to the value, where m is greater than 1 to increase the contrast, and less than 1 to decrease it.

3.1.2. Demonstration

ContentsUpPrevNext
Do it with the GIMP → LUT (Look-Up Tables Effects) → Brightness and Contrast → Demonstration

Demo

  2. Show its histogram ( <Image> -> Image -> "Histogram..." )
  3. Close the histogram.
  4. Access the Brightness and Contrast dialog ( <Image> - Image -> Colors -> "Brightness-Contrast..." )
  5. Modify the Brightness and Contrast in some way.

    Note: the entire modified image is displayed in real-time because Brightness and Contrast changes are so easy to calculate.
  6. Open the Histogram dialog again to show its affects.
  7. Repeat if needed.

3.2. Gamma Correction

ContentsUpPrevNext
Do it with the GIMP → LUT (Look-Up Tables Effects) → Gamma Correction

Question

How can we brighten or darken an image without causing colour values to overflow?

3.2.1. Concept

ContentsUpPrevNext
Do it with the GIMP → LUT (Look-Up Tables Effects) → Gamma Correction → Concept

If we assume that the maximal colour value is 1 and the minimal is 0, then by applying a function of x ** (1/gamma) to every pixel we will modify the histogram while not overflowing from the image boundaries (which cannot be said on brightness and contrast manipulations).

The more positive the Gamma is the brighter the resultant image will be.

3.2.2. Demonstration

ContentsUpPrevNext
Do it with the GIMP → LUT (Look-Up Tables Effects) → Gamma Correction → Demonstration

Demo

  1. Open the A-Little-Exercise-3.jpg image.
  2. Run <Image> -> Image -> Colors -> "Levels..." .
  3. Raise the value of the middle input box of "Input", and show the difference.
  4. Try to set the brightness/contrast on the original image and see that it doesn't work well.

3.3. Equalize

ContentsUpPrevNext
Do it with the GIMP → LUT (Look-Up Tables Effects) → Equalize

The "Equalize" operation constructs an appropriate Look-Up Table so that the histogram of the resultant image will be more or less a constant function.

Low colour values will remain lower than higher ones, but the spread of them will change.

Demo

  2. Show its histogram
  3. Equalize it using <Image> -> Image -> Colors -> Auto -> Equalize
  4. Looks Better, doesn't it?
  5. Show its histogram.

4.1. Blur

ContentsUpPrevNext
Do it with the GIMP → Basic Areal Effects → Blur

Question: How can we blur?

  • Can you come up with a methodical filter that when applied on an image, the image will appear blurred?

4.1.1. Answer to how one can blur an image

ContentsUpPrevNext
Do it with the GIMP → Basic Areal Effects → Blur → Answer to how one can blur an image
  • A blurred image has less abrupt areal changes than the original.
  • Thus, we can blur by averaging every pixel with some of the energy of its neighbours.
  • A blurring acts as a low-pass filter by removing high areal frequencies which cause abrupt changes.

4.1.2. Demo of a simple blur

ContentsUpPrevNext
Do it with the GIMP → Basic Areal Effects → Blur → Demo of a simple blur

Demo

  1. Open lena.png.
  2. Select <Image> -> Filters -> Generic -> "Convolution Matrix..."
  3. Set the 9 middle squares to 1, the other squares to 0 and the divisor to 9.
    (What we are doing is replacing every pixel with the average of its 3*3 neighbourhood.)
  4. Press the "OK" button.
  5. Repeat several times with Alt+F (or <Image> -> Filters -> "Repeat Last" ) to strengthen the effect.

4.1.3. Gaussian Blur

ContentsUpPrevNext
Do it with the GIMP → Basic Areal Effects → Blur → Gaussian Blur
  • Gaussian Blur is a blur that uses an elliptical Gaussian as its kernel.
  • Why Gaussian?
  • If we apply a finite blur enough times, then by virtue of the Central Limit Theorem, we will get a Gaussian-like blur.

4.1.3.1. IIR vs. RLE Gaussian Blurs

ContentsUpPrevNext
Do it with the GIMP → Basic Areal Effects → Blur → Gaussian Blur → IIR vs. RLE Gaussian Blurs
  • GIMP supports two implementations of Gaussian Blur: IIR G.B. and RLE G.B.
  • They both produce the same results, but each one can be faster in some cases.

RLE Gaussian Blur

  • RLE stands for run-length encoding
  • RLE Gaussian Blur is best used on computer-generated images or those with large areas of constant intensity.

IIR Gaussian Blur

  • IIR stands for "Infinite Impulse Response".
  • This blur works best for large radius values and for images which are not computer generated.

4.1.3.2. Demo of IIR Gauss. Blur

ContentsUpPrevNext
Do it with the GIMP → Basic Areal Effects → Blur → Gaussian Blur → Demo of IIR Gauss. Blur

Demo

  1. Open lena.png.
  2. Select <Image> -> Filters -> Blur -> "Gaussian Blur (IIR)..."
  3. Select radii of x = 5 and y = 5.
  4. Press the "Apply" Button.
  5. Optionally select different images or radii and play with it.

4.1.3.3. Demo of RLE Gauss. Blur

ContentsUpPrevNext
Do it with the GIMP → Basic Areal Effects → Blur → Gaussian Blur → Demo of RLE Gauss. Blur

Demo

  1. Create a blank white image.
  2. Place a black text caption on it using the text tool.
  3. Select <Image> -> Filters -> Blur -> "Gaussian Blur (RLE)..."
  4. Select radii of x = 5 and y = 5.
  5. Press the "Apply" Button.

5.1.1. Interpolation

ContentsUpPrevNext

ContentsUpPrevNext

Demo

8. Scripting the GIMP

ContentsUpPrevNext
Do it with the GIMP → Scripting the GIMP
  • Gimp was designed from the ground's up to be modular and extensible.
  • Therefore, it has an extensive support for scripting.
  • There are in fact several options to script it. Some of them will only be reviewed here and will not be demonstrated.

8.1. The Procedural Database

ContentsUpPrevNext
Do it with the GIMP → Scripting the GIMP → The Procedural Database
  • Gimp contains a database of named procedures.
  • This database is flat, so care must be taken to maintain namespace integrity
  • Scripting extensions can interact with the GIMP by accessing the functions through their names in the database.
  • This list can be viewed by accessing: <Toolbox> -> Xtns -> "DB Browser..."
  • It is evident that there are several classes of functions which are distinguished by their prefix.

8.2. Script-Fu Scripting

ContentsUpPrevNext
Do it with the GIMP → Scripting the GIMP → Script-Fu Scripting
  • Script-Fu enables to write GIMP Scripts and PDB entries in Scheme.
  • The Scheme scripts reside in ~/.gimp-1.2/scripts
  • Whenever changed they need to be refreshed by selecting <Toolbox> -> Xtns -> Script-Fu -> Refresh
  • Otherwise, GIMP sports an interactive Script-Fu Console.

8.2.1. Interactive Demo

ContentsUpPrevNext
Do it with the GIMP → Scripting the GIMP → Script-Fu Scripting → Interactive Demo

Demo

  1. Invoke the console by selecting: <Toolbox> -> Xtns -> Script-Fu -> Console .
  2. Close all the images.
  3. Create a new image with the background as its filling.
  4. At the command prompt of the console, type (gimp-image-list) and press enter.
  5. Record the index of the image. (it's inside the inner parenthesis)
  6. Type (gimp-image-active-drawable $image) and record the drawable number. (In the GIMP internals, a drawable is a base class for layers and such)
  7. Type (gimp-rect-select $image 10 10 50 80 ADD 0 0) and press enter.
  8. Type (gimp-rect-select $image 40 70 50 80 ADD 0 0) and press enter.
  9. Type (gimp-bucket-fill $drawable FG-BUCKET-FILL NORMAL 100 0 0 0 0) and press enter.
  10. Note: Consult the DB browser for information regarding what every function and parameter here do.

8.2.2. Generating a Logo

ContentsUpPrevNext
Do it with the GIMP → Scripting the GIMP → Script-Fu Scripting → Generating a Logo
  • Now, let's see some Scheme Scripts in action.

Demo

  1. Select <Toolbox> -> Xtns -> Script-Fu -> Logos -> "Bovination..."
  2. Press the button next to "Font:" and select a nice scalable font.
  3. Press OK.
  4. And you have a full-fledged logo.
  • The source code of this script can be seen in /usr/share/gimp/1.2/scripts/bovinated-logo.scm.
  • You can play with the other scripts in the Xtns -> Script-Fu -> Logos menu and view their source code. (my personal favourite is Frosty, which is quite heavy on CPU and time).

8.3. Gimp-Perl Scripting

ContentsUpPrevNext
Do it with the GIMP → Scripting the GIMP → Gimp-Perl Scripting
  • Gimp-Perl scripts are technically plug-ins, which are written in Perl.
  • Inside such a plug-in one can do everything that can be done in Perl - e.g: opening files, sockets, fork()ing and executing other programs and GUI code.
  • Gimp-Perl is not available for Win32 at the time of this writing, and also has a more picky installation process.
  • Gimp-Perl has an extension called Gimp::Pixel that enables to write plug-ins using the Perl Data Language (=PDL). This is, in fact, the GIMP's answer to the Image Toolbox of Matlab.

8.4. GIMP (C-based) Plug-ins

ContentsUpPrevNext
Do it with the GIMP → Scripting the GIMP → GIMP (C-based) Plug-ins
  • If all else fail, one can write a plug-in in C.
  • You can find some documentation about it here: http://gimp-plug-ins.sourceforge.net/
  • Plug-ins can access the GIMP's PDB, so they can invoke other plug-ins (and scripts !).
